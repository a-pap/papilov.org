*Эскиз, не гайд. Git-репозиторий плюс один воркер — достаточно, чтобы у Claude появилась память, переживающая диалог.*

Установка состоит из трёх частей и намеренно маленькая. Приватный git-репозиторий держит короткие Markdown-заметки — крошечный загрузчик в корне, кросс-доменный статус-файл до шестидесяти строк, и файл правил для поведения, которое не хочется переобъяснять каждую сессию. Один файл Cloudflare Worker говорит по Model Context Protocol на `/mcp`, опираясь на D1 для структурированных строк и R2 для эпизодических бэкапов. Два session-хука Claude Code связывают всё: `SessionStart` делает `git pull`, чтобы первый тёрн любой сессии видел свежие данные; `SessionEnd` пишет строку использования через инструмент `auto_log`. Опциональные надстройки — capture-бот, ночная консолидация, пара managed-коннекторов — каждая ~пятнадцать минут, независимы от ядра.

Четыре инварианта превращают это из игрушки в то, чему можно доверять. Держать загрузчик маленьким, чтобы стартовая цена оставалась в пределах пары тысяч токенов. Делать роутинг по теме детерминированным — одна тема живёт в одном файле, никогда в двух. Считать закоммиченный Markdown сильнее, чем всё, что в чат-логах, когда факты расходятся. Никогда не класть секреты в репо; локальный pre-commit scanner и серверный lint гарантируют, что оплошность не доберётся до публичного интернета.

Смежные инструменты, если хочется построить своё: сам [Model Context Protocol](https://modelcontextprotocol.io/); managed-альтернативы [mem0](https://mem0.ai/) и [Letta](https://www.letta.com/); CLI `llm` Саймона Уиллисона и [Datasette](https://datasette.io/) для школы &quot;файл как примитив&quot;. На этой странице нечего вызывать — код приватный, эндпоинты не опубликованы. Паттерн и есть суть.
